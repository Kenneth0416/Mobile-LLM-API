# Android LLM API Design

## Goal

Build and install an Android app on the connected Sony XQ-BE72 that runs a local LiteRT-LM language model and exposes a small HTTP API so the Mac can call the phone's LLM compute during app development.

## Device Facts

- Device: Sony XQ-BE72 / Xperia PRO-I
- Android: 13, API 33
- ABI: arm64-v8a
- SoC: QTI SM8350 / Snapdragon 888
- RAM: about 11.4 GB total, about 7 GB available during inspection
- Storage: about 453 GB available under user data

## Approach

Use Google's LiteRT-LM Android AAR (`com.google.ai.edge.litertlm:litertlm-android:0.13.1`) instead of compiling native inference code. The machine has Android SDK/build-tools and Android Studio JBR, but no NDK/CMake, so the AAR path is the fastest reliable route to a directly installable APK.

The app runs an Android service that starts an embedded HTTP server on port 18080. The Mac accesses it with `adb forward tcp:18080 tcp:18080`, then sends OpenAI-style JSON to `POST /v1/chat/completions`.

## Model Choice

Default model:

- `litert-community/Qwen3-0.6B`
- File: `qwen3_0_6b_mixed_int4.litertlm`
- Size: about 497,664,000 bytes
- License: Apache-2.0
- Gate: none

This is a conservative default for Snapdragon 888 and 11 GB RAM. It keeps the first install/test cycle realistic while leaving room to add larger 1B/2B models later. Gemma models remain candidates, but many are gated by license acceptance, so they are not the default for automated installation.

## API

- `GET /health` returns service, device, active model, and runner status.
- `GET /v1/models` returns the local model catalog and selected recommendation.
- `POST /v1/chat/completions` accepts `{ "messages": [{ "role": "user", "content": "..." }] }` and returns a minimal OpenAI-compatible chat completion JSON object.

Streaming is intentionally out of scope for the first working build. The first target is a verified local non-streaming response from the phone.

## Verification

Completion requires all of the following evidence:

- Unit tests pass for JSON parsing, response formatting, model recommendation, and HTTP handler behavior.
- APK builds successfully.
- Qwen3 0.6B LiteRT-LM model is present on the phone under the app external files model directory.
- APK is installed on the connected phone.
- App service is started.
- `adb forward` maps Mac port 18080 to device port 18080.
- A Mac-side HTTP call to `/v1/chat/completions` returns an assistant message generated by the local phone model.
